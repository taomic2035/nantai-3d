# Viewer 内嵌模式无法使用坐标传送

## Bug 诊断胶囊

| 栏位 | 内容 |
|---|---|
| **1. 现象** | Viewer 的帮助栏声明按 `G` 可传送到任意 ENU 坐标。实际在内嵌 Viewer 中按 `G` 没有出现可见输入界面，坐标与 chunk 均不变化；同一画布上的 `F` 自由视角快捷键正常。 |
| **2. 证据** | 2026-07-19 在 `http://127.0.0.1:8767/web/viewer/` 的真实 Spark 点云模式稳定复现。按 `F` 后 HUD 从“环绕”变为“自由”；按 `G` 后日志明确报 `Error: prompt() is not supported`。源码 `web/viewer/main.js` 的 `G` 分支唯一动作是调用 `window.prompt()`。 |
| **3. 根因** | 传送交互依赖浏览器原生模态框，而 Viewer 会运行在受宿主约束的内嵌页面中；该环境没有提供可用的原生 `prompt()` 交互面。坐标解析和 `moveCameraTo()` 均已存在，失效点在输入边界。 |
| **4. 诊断策略** | 对照同一 `keydown` 监听中的 `F` 工作路径，确认键盘事件可达；再从 `G` 逆向追踪到唯一的 `window.prompt()` 输入边界，保持坐标解析、ENU→Three 映射和 chunk 调度不变。 |
| **5. 超时策略** | 若页面内对话框仍无法打开，先独立验证 `showModal()` 和焦点管理；若对话框可提交但坐标不变，再转查 `parseEnuText()` 与 `moveCameraTo()`，不同时改动加载器。 |
| **6. 预警策略** | 静态合约通过但实际页面没有 `dialog`、提交后 HUD 未显示目标 ENU，或负坐标 chunk 没有加载，均说明修复未完成。 |
| **7. 用户可见交互修正** | `G` 打开页面内“传送到任意坐标”面板；预填当前 ENU，输入错误就地说明且不移动，合法提交后关闭面板、移动相机并立即加载目标分块。 |
| **8. 验收** | Viewer DOM/运行时合约先因缺少页面内面板按预期变红；修复后定向 30 tests 与 Viewer 全量测试通过。真实浏览器非法值保持面板开启并显示错误；合法输入 `-1200,1800,120` 后 HUD 显示该 ENU、当前 chunk `(-6,9)`，目标分块最终空闲；自由视角正反拖拽改变可见方向，六种天气均切换成功。全新标签的 warning/error 日志为空。 |

## 五件套

### 1. 报告人

Codex 在完整目标的真实 Viewer 浏览器验收中发现。

### 2. 复现步骤

1. 启动 Studio server，打开 Viewer 并切换到点云 / 高斯模式。
2. 聚焦“3D 场景画布”，按 `F`，确认 HUD 能切换到自由视角。
3. 再按 `G`。
4. 观察没有坐标输入界面，HUD 坐标和当前 chunk 不变。

期望：能在内嵌页面输入任意 ENU 坐标并传送。实际：依赖宿主没有提供的原生
`prompt()`，用户无法完成输入。

### 3. 根因分析

`keydown` 数据流已经由 `F` 的成功切换证明可达。`G` 分支没有其它条件或状态门控，
唯一用户输入来自 `window.prompt()`。坐标解析、坐标系转换、相机移动和传送后 chunk
调度均位于其下游，尚未获得输入。根因因此是宿主敏感的原生模态交互边界，而不是
ENU 公式或按需渲染端点。

### 4. 修复方案

用文档内的可访问 `dialog + form` 代替 `window.prompt()`，保留既有
`parseEnuText()`、`worldToThree()` 与 `moveCameraTo()`。提交时 fail-closed：
非法值显示中文错误、保持面板开启且绝不移动相机；合法值才移动并关闭。

### 5. 验证方式

- `web/viewer/index-contract.test.mjs`：页面内传送控件、运行时接线、无
  `window.prompt()` 回归。
- `web/viewer/camera-pose.test.mjs`：ENU 文本解析继续覆盖正负数与错误输入。
- `node --test web/viewer/*.test.mjs`：Viewer 全量回归。
- 内嵌浏览器：真实负坐标传送、目标 chunk 加载、自由视角旋转和天气切换。

最终浏览器证据：非法 `1,2` 显示“需要 3 段”且不关闭面板；合法
`-1200, 1800, 120` 后 HUD 变为该坐标和 chunk `(-6,9)`，36 个缓存槽完成调度并回到
“空闲”。同一位置的水平正反拖拽呈现不同视野；晴、阴、雨、雪、雾、夜六项 HUD
均与选择一致。全新标签复测后的 warning/error 日志为 `[]`。
